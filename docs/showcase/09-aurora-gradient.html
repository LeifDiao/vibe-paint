Building Design Systems That Scale Across Teams

SC
Sarah Chen · Mar 20, 2026 · 8 min read

When we started building our component library, we made a choice that would define everything: tokens first, components second. Every color, every spacing value, every shadow — codified as CSS custom properties before a single button was designed.

The result? A system where changing --color-primary cascades through 200+ components instantly. No find-and-replace. No broken states.

"The best design system is one you forget you're using."

— Our internal design principle

The Token Architecture

We structured tokens in three layers: primitive (raw values), semantic (purpose-driven aliases), and component (scoped to specific UI elements). This layering means you can swap an entire theme by only touching the semantic layer.
